Blogging for the first time

Assaf Schwartz
2021-02-21
signal, travel

2020 has been crazy.

Israel has gone through 3 lockdowns, with the latest spanning for about a month.
My SO and I planned a 3 day getaway (1 per lockdown?) up to the north of the country.

Today she asked me, “do we have some automated system for making orders?”
I was confused. “what for?” I asked.

As vaccines are getting more common, some activities and parks are slowly becoming available again. Mt. Hermon is located in the north of Israel, and is usually the go-to location for snow seekers. If we want to see it, we’ll have to book in advance.
It seems like orders are only availble 1 week in advance, and instead of refreshing the website until our thumbs fall off, I decided to pick up on the “offer”, and try automate it.

Figure out when vouchers are avaialble

To get the vouchers, you would go to https://hermon.pres.global/vouchers, see if the date you are interested in is avaialble, and pick it up from there.

The first thing I did was try to see if I can identify avaialblility for specific dates. (thanks Chrome Dev Tools).

OK, I thought to myself - Beautiful Soup is probably going to help, I’ll install it and see if I can yank out the relevant elements.
curling the address did me little good, as all I got back were a bunch of CSS with no concrete data, nothing like what I’ve seen in Chrome.

Hhmm… I need a browser to render this for me. I never tried Selelium, but why not?
So now we have the following: Vouchers website → Selelium → Beautiful Soup and hold and behold, we can gain some info about relevant dates.

Getting notified

Our households has a NAS (Linux), and several Macs.
For Selenium, I figured a system with proper Window Manager is required, so the Mac Mini was chosen.

As we are both Android users, I needed to find a way to push notifications from the Mac, which is running the scaping script, to our non iOS devices.

SMS? Voted out because all of those services seemed like an overhead…(the setup alone!)
Email server? Setting it up on the NAS was even more painful!

And then it occured to me. I recently switched to Signal. I Googled ‘signal cli’, and found some really interesting things.

To be more specific signal-cli-rest-ap, this allows you to interact with Signal using RESTful APIs.

Setting up signal-cli-rest-api

Luckily, the NAS has docker installed. So after setting it up, making sure I expose port 8080, I tried registering the CLI as a valid client. I should have read the docs more closely. Using my number for registration got my phone kicked off Signal, as you can register a number with a single primary device.

Back to the docs, it seems that you can also link devices (pretty much what you would do with the Signal Desktop client).
This is actually a feature of signal-cli, so got myself into the container and ran signal-cli link -n "<device name>"

This spits out a link in the form of tsdevice:/... which you can use to generate a QR code.
Next, link the new device the same way you would the Desktop app.

Now, we can use signal-cli-rest-api to send messages over Signal.
It’s as easy as POSTing to the endpoint.
